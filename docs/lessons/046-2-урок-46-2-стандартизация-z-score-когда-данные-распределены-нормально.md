# Урок 46/2. Стандартизация (Z-score): когда данные распределены нормально

### RU
Стандартизация — это превращение признака в такой, у которого **среднее = 0, а стандартное отклонение = 1**.

**Формула (простая):**

```
z = (x - mean) / std
```

Где:
- x — исходное значение
- mean — среднее по всем значениям
- std — стандартное отклонение

**Что происходит с данными:**
После стандартизации:
- Значения выше среднего — положительные
- Значения ниже среднего — отрицательные
- 68% значений лежат между -1 и 1
- 95% значений лежат между -2 и 2

**Пример:**
Рост учеников в классе:
- Средний рост = 165 см
- Стандартное отклонение = 10 см
- Ученик A = 175 см → z = (175-165)/10 = 1.0
- Ученик B = 150 см → z = (150-165)/10 = -1.5

Теперь рост ученика A — «на 1 стандартное отклонение выше среднего».

**Когда использовать:**
✅ Данные имеют примерно нормальное распределение (колокол)
✅ Данные уже без сильных выбросов
✅ Нужно сохранить информацию о распределении

**Реализация в Python:**
```python
from sklearn.preprocessing import StandardScaler

scaler = StandardScaler()
scaled_data = scaler.fit_transform(data)
# -> mean = 0, std = 1
```

**Важно:** fit только на тренировочных данных! Потом transform и на тренировочных, и на тестовых. Нельзя fit на всём датасете — это утечка данных.

**Типичная ошибка:** Стандартизировать данные с сильными выбросами. Выбросы искажают среднее и std, и после стандартизации нормальные точки всё равно будут неравномерны.

**Короткое правило:** Стандартизация — для нормальных распределений. Выбросы сначала чистим, потом стандартизируем.

---
### EN
Z-score standardization transforms data to mean=0 and std=1. Use when data is roughly normally distributed without strong outliers.

```python
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
scaled = scaler.fit_transform(data)
```

Always fit on training data only, then transform both train and test sets.
