# ML. Урок 52/3 — Feature engineering
## Ошибки, автоматизация и практика

Feature engineering — не только создание признаков, но и умение не навредить.

### Главные ошибки

**Ошибка 1. Утечка данных (data leakage)**

Самая опасная ошибка. Ты создаёшь признак, который использует информацию из будущего.

Пример. Ты предсказываешь, уволится ли сотрудник. Ты создаёшь признак «средняя зарплата за весь период работы». Но когда модель будет предсказывать для нового сотрудника — у тебя нет данных за «весь период», есть только за прошлое.

**Как не допустить:** все статистики (среднее, максимум и т.д.) считай ТОЛЬКО по прошлым данным. Никогда — по будущим.

**Ошибка 2. Слишком много признаков**

Добавил 100 новых признаков — модель переобучилась. Она запомнила шум, а не закономерности.

**Правило:** каждый новый признак должен иметь обоснование. «Может, пригодится» — плохое обоснование.

**Ошибка 3. Признаки с одинаковой информацией**

Добавил «сумму в рублях», «сумму в долларах», «сумму в евро». Модель получила три одинаковых признака (они коррелируют). Это не помогает, а вредит.

**Проверка:** если два признака сильно коррелируют (коэффициент > 0,9) — оставь один.

### Автоматизация feature engineering

Не всё нужно делать руками. Есть инструменты:

- **Polynomial features** — автоматически создаёт перемножения признаков (x₁, x₂, x₁², x₂², x₁ × x₂)
- **Date decomposition** — разбивка даты автоматически
- **Text statistics** — длина текста, количество слов, количество заглавных букв

**Важно:** автоматика — это старт, а не финиш. Лучшие признаки — те, что ты создал, понимая бизнес.

### Практический совет

Feature engineering занимает 70% времени в реальном ML-проекте. Не торопись. Каждый новый признак проверяй:
1. В нём нет информации из будущего?
2. Он не дублирует другой признак?
3. Он имеет смысл для бизнеса?

### Живой пример

Ты работаешь с данными доставки. Есть столбец «время заказа».

Плохо: оставить как есть (модель не понимает даты).

Хорошо:
- Час заказа (ночью заказывают реже, но быстрее доставляют — мало заказов)
- День недели (в пятницу — пик, в понедельник — спад)
- Выходной или будний (в выходные другие паттерны)
- Сколько времени прошло с последнего заказа (лояльность)

Из 1 столбца сделал 4 осмысленных признака. Модель стала видеть реальность.

### Главная мысль

Feature engineering — это когда ты помогаешь модели увидеть то, что очевидно для человека, но неочевидно для чисел. Из даты — день недели. Из суммы — частоту. Из транзакций — привычки.
