# Урок 45. Линейная алгебра для ML — минимум, который реально нужен

📊 **Урок 45. Линейная алгебра для ML — минимум, который реально нужен**

Признайся: когда слышишь «линейная алгебра», хочется закрыть вкладку? Я тебя понимаю. Большинство курсов начинают с определителей, собственных векторов и прочей теории, которую ты забудешь через неделю.

А теперь правда: для работы с ML тебе нужно ровно **три** понятия. Всё. Остальное досидишь, когда упрёшься.

---

**1. Векторы — это просто списки чисел**

Вектор — не стрелочка в пространстве. В ML вектор — это **одна строка твоих данных**.

Пример:
`[25, 1, 45000, 0]` — это вектор одного клиента:
— возраст 25
— пол мужской (1)
— доход 45 000
— покупал ли (0 = нет)

Каждый столбец в таблице — это **признак**. Каждая строка — это **вектор признаков**.
Вектор = одна запись. Всё просто.

**Главная операция — скалярное произведение:**

`[a₁, a₂, a₃] · [b₁, b₂, b₃] = a₁×b₁ + a₂×b₂ + a₃×b₃`

Звучит страшно? А на деле это то, что делает линейная регрессия:
- У тебя есть веса модели: `[w₁, w₂, w₃]`
- И признаки объекта: `[x₁, x₂, x₃]`
- Предсказание = `w₁·x₁ + w₂·x₂ + w₃·x₃` + смещение

Всё. Это называется скалярное произведение. Линейная регрессия — это просто скалярное произведение весов на признаки.

**Пример из жизни:**
Ты предсказываешь цену квартиры. Признаки: площадь (50), этаж (5), удалённость от метро (0.5). Веса модели: [2000, 1000, -3000]. Предсказание: 50×2000 + 5×1000 + 0.5×(-3000) = 100000 + 5000 - 1500 = **103 500 руб**.

Никакой магии. Просто умножение и сложение.

---

**2. Матрицы — это таблицы чисел**

Матрица — это просто несколько векторов, собранных вместе.
Строки = объекты (клиенты, товары, фотки).
Столбцы = признаки (возраст, доход, цвет, размер).

Размерность матрицы: `(количество строк) × (количество столбцов)`.
Матрица 1000×5 = 1000 клиентов, 5 признаков.

**Умножение матриц — самая важная операция в ML**

Выглядит сложно, но смысл прост: **преобразовать данные из одного пространства в другое**.

Когда нейросеть делает слой:
`h = X · W + b`
— это умножение матрицы входных данных на матрицу весов.
Вход был 10 признаков, стало 64 нейрона. Данные «переехали» в новое пространство.

**Матричное умножение в Python:**
```python
import numpy as np
X = np.array([[1, 2], [3, 4]])  # матрица 2×2
W = np.array([[0.5], [1.5]])     # матрица 2×1
result = X @ W                    # умножение → матрица 2×1
# или X.dot(W)
```

Зачем это нужно? Вместо того чтобы умножать каждый вектор отдельно (цикл for), ты умножаешь всю матрицу сразу — в 100500 раз быстрее.

---

**3. Евклидово расстояние — как измерять похожесть**

Если у тебя два вектора, как понять, похожи они или нет?

**Евклидово расстояние** — это обычное расстояние между точками, как в школе: √((x₁−y₁)² + (x₂−y₂)² + ...)

Где это нужно в ML:
- **К-ближайших соседей (KNN):** ищем похожие объекты по минимальному расстоянию
- **Кластеризация (K-means):** группируем близкие точки
- **Рекомендательные системы:** ищем пользователей с похожими векторами предпочтений
- **Embeddings:** в NLP слова, фразы и тексты превращаются в векторы, и расстояние показывает смысловую близость

**Живой пример:**
У тебя три фильма в векторном пространстве жанров:
- «Матрица» = [0.9, 0.1, 0.2] (фантастика, экшн, драма)
- «Терминатор» = [0.85, 0.15, 0.1] (очень похож)
- «Титаник» = [0.0, 0.1, 0.95] (совсем другой)

Расстояние между «Матрицей» и «Терминатором» — маленькое. Между «Матрицей» и «Титаником» — большое. Система рекомендует то, что ближе.

---

**Зачем это тебе как аналитику?**

Ты не будешь вручную считать собственные векторы. Но ты будешь:

1. **Готовить данные** — каждую таблицу можно и нужно мыслить как матрицу. Строки = объекты, столбцы = признаки.
2. **Понимать, что делает модель** — линейная регрессия, PCA, нейросети, embeddings — всё это операции над матрицами и векторами.
3. **Отлаживать** — когда модель выдаёт чушь, понимание размерностей (shape) матриц спасёт часы дебага. Самая частая ошибка: не сошлись размерности при умножении.
4. **Читать документацию** — `X.shape`, `reshape()`, `@`, `.T` — ты будешь видеть это каждый день.

**Проверочная задачка:**
У тебя матрица признаков размером (500, 10) — 500 клиентов, 10 признаков. Ты хочешь умножить её на матрицу весов (10, 3) — преобразовать 10 признаков в 3 новых. Какой размерности будет результат?

(Ответ: 500×3 — 500 строк, 3 столбца. Строки остались, изменились только признаки.) ✅

---

**Короткое резюме:**

1. **Вектор** = одна строка данных (один объект)
2. **Матрица** = таблица данных (много объектов)
3. **Скалярное произведение** = то, что делает линейная регрессия
4. **Умножение матриц** = преобразование данных в новое пространство
5. **Евклидово расстояние** = мера похожести объектов

Больше для старта не нужно. Остальное — когда упрёшься. 🎯
