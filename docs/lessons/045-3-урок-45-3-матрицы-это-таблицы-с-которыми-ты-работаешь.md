# Урок 45/3. Матрицы — это таблицы, с которыми ты работаешь

📊 **Урок 45/3. Матрицы — это таблицы, с которыми ты работаешь**

Ты работаешь с матрицами каждый день. Просто не называл их так.

---

**Что такое матрица в ML?**

Матрица — это прямоугольная таблица чисел. В ML матрица — это **твой датасет**.

Строки = объекты (клиенты, транзакции, изображения).
Столбцы = признаки (возраст, доход, цвет, размер).

Матрица размером **(1000 × 5)** = 1000 строк и 5 столбцов.
— 1000 клиентов
— 5 признаков у каждого

Всё. Не нужно представлять ничего сложнее Excel-таблицы.

---

**Как обозначают матрицы**

Обычно матрицы обозначают большими буквами.

`X` — матрица признаков (features). Чаще всего так.
`W` — матрица весов (weights).
`Y` — матрица целевых значений (target).

Размерность матрицы записывают так: **(n × m)**.
n = количество строк.
m = количество столбцов.

Если n больше m — объектов много, признаков мало. Так бывает чаще всего.

---

**Зачем нужно понятие матрицы?**

**1. Чтобы мыслить структурированно.**

Когда ты берёшь новый датасет, первое, что нужно понять — его размерность (shape).
— Сколько у нас данных? (строки)
— Сколько признаков? (столбцы)
— Есть ли пропуски?

Ты уже видишь это в Excel: строка 1 из 1000, столбец A–E. Поздравляю, ты работаешь с матрицей.

**2. Чтобы понимать, как модель «видит» данные.**

Модель не понимает ни названия столбцов, ни смысла данных. Она видит просто матрицу чисел. Красивый график, подписи, контекст — это для человека. Модели нужна только прямоугольная таблица.

**3. Чтобы не путаться в размерностях (это спасёт часы дебага).**

Самая частая ошибка новичков: размерности не совпадают.
— У тебя матрица (100, 10), а веса (5, 1). Умножить не получится.
— Внутренние размерности должны совпадать.

Простое правило: количество столбцов первой матрицы должно равняться количеству строк второй.

---

**Специальные виды матриц, которые встретятся:**

**Матрица с одной колонкой — это вектор-столбец.**
Размерность (n × 1). Например, целевая переменная — все цены квартир в одном столбце.

**Матрица с одной строкой — это вектор-строка.**
Размерность (1 × m). Например, один новый объект для предсказания.

**Квадратная матрица.**
Количество строк = количество столбцов. Встречается в ковариационных матрицах, корреляции и некоторых продвинутых методах.

**Единичная матрица.**
На главной диагонали — единицы, остальное — нули. Это «единица» в мире матриц. При умножении на неё матрица не меняется.

---

**Матрица в numpy (просто чтобы знать):**

Когда ты в Python видишь:
```python
X.shape  # выводит что-то вроде (500, 10)
```
— это просто про размер твоей матрицы. 500 строк, 10 столбцов.

`.shape` ты будешь писать постоянно. Запомни это слово.

---

**Резюме:**

1. Матрица = твой датасет. Строки = объекты, столбцы = признаки
2. Размерность записывается как (строки × столбцы)
3. Модель видит только матрицу чисел — без названий и контекста
4. `.shape` — главная команда, которая спасает от ошибок
5. Не путай размерности — это самая частая ошибка в ML

---

**Проверь себя:**

У тебя данные по 300 товарам: цена, вес, рейтинг, количество отзывов.

Какая размерность у матрицы признаков?

(Ответ: 300 × 4. 300 строк — товары, 4 столбца — признаки.) ✅
