# Урок 46/5. Как не ошибиться: главная ловушка масштабирования

### RU
Самая дорогая ошибка в масштабировании — **утечка данных (data leakage)**.

**Как выглядит:**
```python
# ❌ НЕПРАВИЛЬНО: fit на всех данных
scaler = StandardScaler()
scaler.fit(X_full)  # fit на всём датасете
X_train_scaled = scaler.transform(X_train)
X_test_scaled = scaler.transform(X_test)
```

**Почему это плохо:**
Когда ты fit на всём датасете, модель уже «знает» статистику тестовых данных (среднее, std, min, max). Это нечестно — в реальности модель будет видеть только то, что есть в обучающих данных.

**Правильно:**
```python
# ✅ ПРАВИЛЬНО: fit только на train
from sklearn.model_selection import train_test_split

X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train)  # fit + transform для train
X_test_scaled = scaler.transform(X_test)         # только transform для test
```

**Вторая ловушка — нестандартизированные целевые переменные**
Иногда нужно масштабировать и Y (например, в регрессии с огромными числами). Для этого — TransformedTargetRegressor.

**Третья ловушка — забыть про pipeline**
Когда масштабирование и модель — отдельные шаги, легко ошибиться. Используй Pipeline:

```python
from sklearn.pipeline import make_pipeline
from sklearn.neighbors import KNeighborsClassifier

pipe = make_pipeline(StandardScaler(), KNeighborsClassifier())
pipe.fit(X_train, y_train)  # scaler.fit → model.fit в одном шаге
pipe.score(X_test, y_test)  # scaler.transform → model.predict
```

Pipeline гарантирует, что масштабирование всегда применяется на train как fit_test.

**Четвёртая ловушка — масштабировать категориальные признаки**
One-hot признаки уже в [0, 1]. MinMaxScaler не навредит, но лишняя операция. StandardScaler сделает из (0, 1) -> (отрицательные, положительные) — зачем?

**Пятая ловушка — не сохранить scaler для продакшна**
```python
import joblib
# Сохраняем
joblib.dump(scaler, 'my_scaler.pkl')
# В продакшне
scaler = joblib.load('my_scaler.pkl')
new_scaled = scaler.transform(new_data)
```
Никогда не пересчитывай min/max/mean на новых данных в продакшне — это ломает всю модель.

**Сводка: что запомнить**
1. ✅ Fit — только на train
2. ✅ Pipeline — твой друг
3. ✅ Scalers для продакшна — сохраняй
4. ✅ Выбор scaler — по распределению данных
5. ✅ Категориальные признаки не трогай (если не надо)

---
### EN
**Golden rule:** Fit the scaler on TRAINING data ONLY. Apply transform to both train and test. Use Pipeline to avoid mistakes. Save the scaler for production — never refit on new data.

```python
pipe = make_pipeline(StandardScaler(), KNeighborsClassifier())
pipe.fit(X_train, y_train)
```
