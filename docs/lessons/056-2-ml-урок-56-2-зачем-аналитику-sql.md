# ML. Урок 56/2 — Зачем аналитику SQL
## Что именно даёт SQL в реальной работе

Продолжим. SQL — это не просто способ «что-то вытащить из базы». Это навык, который резко усиливает аналитика.

### 1. SQL помогает быстро проверять гипотезы

Аналитика — это не только красивые дашборды и не только модели.

Очень часто аналитика — это цепочка быстрых вопросов:
- правда ли, что новый тариф покупают хуже старого?
- действительно ли пользователи из одного канала отваливаются чаще?
- правда ли, что рост жалоб связан с конкретным продуктом?

Когда ты знаешь SQL, ты можешь сам быстро проверить гипотезу.

Без SQL ты зависишь от чужой скорости.

### 2. SQL помогает собирать данные правильно, а не как попало

Очень опасная ошибка новичка — взять какую-то случайную выгрузку и начать анализировать её как истину.

Но хороший аналитик должен понимать:
- откуда пришли данные
- какие таблицы использованы
- какие фильтры стоят
- что означают поля
- нет ли дублей
- нет ли пропущенных строк из-за неверного соединения

SQL делает тебя не просто потребителем выгрузки, а человеком, который понимает происхождение данных.

А это уже другой уровень работы.

### 3. SQL делает тебя сильнее даже в Python и ML

Парадокс в том, что SQL усиливает даже тех, кто потом работает в pandas, scikit-learn и других инструментах.

Почему?

Потому что до модели сначала нужно дойти.

Нужно:
- собрать правильные данные
- отфильтровать лишнее
- посчитать агрегаты
- соединить таблицы
- подготовить основу датасета

Если ты это умеешь в SQL, ты быстрее доходишь до анализа и моделирования.

### Живой пример 3

Допустим, тебе нужно посчитать средний чек клиентов по городам только за последние 90 дней.

Если у тебя нет SQL, ты можешь попросить сырую выгрузку на миллионы строк, загрузить всё в ноутбук и потом долго чистить.

Если у тебя есть SQL, ты сразу заберёшь только нужное:
- только нужный период
- только нужные поля
- сразу с группировкой
- сразу в компактном виде

Ты экономишь время, память и нервы.

### 4. SQL помогает говорить с инженерами и BI-командой на одном языке

Когда аналитик не знает SQL, он часто формулирует задачу туманно:
- «мне нужны данные по клиентам»
- «а можно выгрузить поведение пользователей»
- «а тут что-то не сходится»

Когда аналитик знает SQL, он думает точнее:
- из какой таблицы
- по какому ключу
- по какому периоду
- с каким фильтром
- с какой агрегацией

И качество коммуникации резко растёт.

### Мини-задача

Представь задачу:
**Нужно найти клиентов, которые ничего не покупали последние 60 дней, но раньше покупали регулярно.**

Попробуй мысленно ответить:
- какие таблицы для этого нужны
- какая дата будет важна
- что значит «регулярно»
- как ты поймёшь, что клиент действительно перестал покупать

Это уже мышление SQL-аналитика, даже без написания запроса.

### Частая ошибка

Новичок часто думает:
**«Сначала выучу Python, а SQL потом как-нибудь сам придёт».**

Обычно не приходит.

Потому что SQL — это не приложение к аналитике. Это один из базовых рабочих инструментов.

### Главное

SQL не делает тебя «только SQL-щиком». Он делает тебя аналитиком, который умеет сам дойти до данных, сам проверить гипотезу и сам собрать основу для сильного анализа.

А значит — делает тебя намного ценнее в реальной работе.
