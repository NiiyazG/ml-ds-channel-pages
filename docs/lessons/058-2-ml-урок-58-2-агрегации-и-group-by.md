# ML. Урок 58/2 — Агрегации и GROUP BY

## Что такое агрегации

Агрегация — это вычисление итога внутри группы или по всей таблице.

Самые важные агрегатные функции:
- `COUNT()` — количество
- `SUM()` — сумма
- `AVG()` — среднее
- `MIN()` — минимум
- `MAX()` — максимум

Именно они чаще всего работают вместе с `GROUP BY`.

## Пример 1 — выручка по товарам

```sql
SELECT product, SUM(amount)
FROM orders
GROUP BY product;
```

Здесь SQL считает, сколько денег принёс каждый товар.

## Пример 2 — средний чек по городам

```sql
SELECT city, AVG(amount)
FROM orders
GROUP BY city;
```

Теперь можно увидеть, в каком городе средний чек выше, а в каком ниже.

## Очень важное правило

Если в запросе есть `GROUP BY`, то в `SELECT` нельзя просто так вставлять случайные поля.

Нужно понимать логику:
- либо поле входит в `GROUP BY`
- либо оно считается через агрегатную функцию

Например, это корректно:

```sql
SELECT city, COUNT(*)
FROM orders
GROUP BY city;
```

А если без логики добавить `order_id`, запрос станет неправильным по смыслу.

## GROUP BY + WHERE

```sql
SELECT city, COUNT(*)
FROM orders
WHERE status = 'completed'
GROUP BY city;
```

Что происходит:
1. сначала берём только завершённые заказы
2. потом группируем их по городам
3. потом считаем количество

## GROUP BY + ORDER BY

```sql
SELECT city, SUM(amount)
FROM orders
GROUP BY city
ORDER BY SUM(amount) DESC;
```

Теперь ты увидишь города в порядке убывания выручки.

## Где новички чаще всего ошибаются

1. Путают строки и группы
2. Пихают в `SELECT` лишние поля
3. Не понимают заранее, по какому признаку группируют и что считают

## Мини-задача

Представь таблицу заказов, где есть:
- город
- сумма заказа
- статус заказа

Попробуй словами описать, как бы ты получил:
1. количество завершённых заказов по городам
2. общую выручку по городам
3. средний чек по городам

## Главное

`GROUP BY` группирует строки по признаку.
А агрегатные функции считают результат внутри этих групп.

Именно здесь SQL начинает работать как настоящий аналитический инструмент.
