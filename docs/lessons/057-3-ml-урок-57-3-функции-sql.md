# ML. Урок 57/3 — Функции SQL
## Чем функция отличается от оператора и как запрос начинает считать

Теперь важный следующий шаг.

Мы уже разобрали, что в SQL есть **операторы**:
- `SELECT`
- `FROM`
- `WHERE`
- `ORDER BY`

Но в SQL есть ещё и **функции**.

Это не одно и то же.

---

## Чем оператор отличается от функции

### Оператор
Оператор задаёт структуру запроса.
Он говорит базе, что делать в целом.

Примеры операторов:
- `SELECT`
- `FROM`
- `WHERE`
- `ORDER BY`

### Функция
Функция выполняет конкретное действие над данными.

Примеры функций:
- `COUNT()` — посчитать количество
- `SUM()` — посчитать сумму
- `AVG()` — посчитать среднее
- `MIN()` — найти минимум
- `MAX()` — найти максимум

Проще говоря:

- **оператор** — это часть конструкции запроса
- **функция** — это действие внутри запроса

---

## Пример 1 — COUNT()

Нужно посчитать, сколько клиентов есть в таблице:

```sql
SELECT COUNT(*)
FROM clients;
```

Что здесь происходит:
- `COUNT(*)` считает количество строк
- `FROM clients` говорит, из какой таблицы считать

---

## Пример 2 — SUM()

Нужно посчитать общую сумму продаж:

```sql
SELECT SUM(amount)
FROM orders;
```

Здесь:
- `SUM(amount)` складывает значения столбца `amount`

---

## Пример 3 — AVG()

Нужно узнать среднюю сумму заказа:

```sql
SELECT AVG(amount)
FROM orders;
```

Здесь:
- `AVG(amount)` считает среднее значение

---

## Пример 4 — MIN() и MAX()

Нужно найти минимальную и максимальную сумму заказа:

```sql
SELECT MIN(amount), MAX(amount)
FROM orders;
```

Здесь:
- `MIN(amount)` ищет минимальное значение
- `MAX(amount)` ищет максимальное значение

---

## Как операторы и функции работают вместе

Вот пример:

```sql
SELECT COUNT(*)
FROM clients
WHERE city = 'Москва';
```

Как это читается:
- `COUNT(*)` — посчитать количество
- `FROM clients` — в таблице клиентов
- `WHERE city = 'Москва'` — только по клиентам из Москвы

То есть функции не живут отдельно.
Они работают внутри общей логики запроса.

---

## Ещё один пример

```sql
SELECT AVG(amount)
FROM orders
WHERE status = 'completed';
```

Что происходит:
- берём только завершённые заказы
- считаем средний чек по ним

Вот тут SQL уже начинает не просто показывать данные, а **сразу считать смысловые показатели**.

---

## Почему это важно аналитику

Потому что аналитик очень часто задаёт именно такие вопросы:
- сколько клиентов?
- какая выручка?
- какой средний чек?
- какой максимум?
- какой минимум?
- сколько оплат было за день?

Именно поэтому функции SQL — это не «дополнение», а один из базовых рабочих инструментов.

---

## Как запрос работает по смыслу

Очень важно научиться читать запрос не как странный код, а как последовательность мысли.

Например:

```sql
SELECT name, total_amount
FROM clients
WHERE city = 'Москва'
ORDER BY total_amount DESC;
```

По смыслу это так:
1. Возьми клиентов
2. Оставь только тех, кто из Москвы
3. Покажи имя и сумму покупок
4. Отсортируй по убыванию суммы

Когда ты начинаешь так читать SQL, он резко становится проще.

---

## Практический вывод

Если ты понял разницу между оператором и функцией, то у тебя уже появляется прочная база.

Ты начинаешь видеть, что SQL — это не беспорядочный набор команд, а логичный конструктор:
- операторы строят каркас запроса
- функции делают вычисления внутри него

## Частая ошибка

Новичок часто смешивает всё в одну кучу:
- операторы
- функции
- условия
- сортировку

Из-за этого SQL кажется тяжёлым.

Но как только ты раскладываешь всё по ролям, он становится намного понятнее.

---

## Главное

**Оператор** задаёт структуру запроса.
**Функция** выполняет вычисление внутри запроса.

И когда ты понимаешь обе роли, SQL становится не страшным кодом, а понятным рабочим инструментом аналитика.
