    ML и анализ данных • Урок 68. Оконные функции в SQL¶

    Был у тебя такой момент: нужно к каждой строке добавить среднее по группе — и ты лепишь подзапрос, джойнишь таблицу саму на себя и проклинаешь всё на свете?

    Оконные функции решают это в одну строчку.

    Разбираем синтаксис по косточкам¶

    Вот как выглядит оконная функция:

    [название_функции] ([выражение]) OVER ([определение_окна])

    Три части:

    1️⃣ Функция — что считаем: SUM, AVG, COUNT, MIN, MAX 2️⃣ Выражение — над чем считаем: (salary), (revenue), (quantity) 3️⃣ Определение окна — в каких границах считаем: OVER (PARTITION BY department) или OVER ()

    Проще всего разобрать на примере.

    Пример 1: таблица без оконных — как есть¶

    Берём обычную таблицу сотрудников:

    SELECT name, department, salary
FROM employees;

    Результат: | name | department | salary | |------|-----------|--------| | Иванов | Бухгалтерия | 70 000 | | Петров | Бухгалтерия | 90 000 | | Сидоров | IT | 120 000 | | Кузнецов | IT | 100 000 | | Смирнова | HR | 60 000 |

    Пять строк. Всё как есть.

    Пример 2: добавляем среднее по отделу — окно в действии¶

    SELECT
  name,
  department,
  salary,
  AVG(salary) OVER (PARTITION BY department) AS avg_dept
FROM employees;

    Результат: | name | department | salary | avg_dept | |------|-----------|--------|----------| | Иванов | Бухгалтерия | 70 000 | 80 000 | | Петров | Бухгалтерия | 90 000 | 80 000 | | Сидоров | IT | 120 000 | 110 000 | | Кузнецов | IT | 100 000 | 110 000 | | Смирнова | HR | 60 000 | 60 000 |

    Смотри, что произошло: — Строки остались. Ничего не схлопнулось. — Появилась новая колонка. В ней — средняя зарплата по тому отделу, в котором работает сотрудник. — PARTITION BY department — это и есть окно. SQL говорит: «Возьми всех, кто в одном отделе, посчитай среднее — и запиши этот результат к каждой строке этого отдела». — Бухгалтерия: (70 000 + 90 000) / 2 = 80 000 → записалось в строки Иванова и Петрова — IT: (120 000 + 100 000) / 2 = 110 000 → записалось в строки Сидорова и Кузнецова — HR: (60 000) / 1 = 60 000 → записалось в строку Смирновой

    Синтаксис подробно¶

    <функция> OVER (
  [PARTITION BY колонка_1, колонка_2, ...]
  [ORDER BY колонка [ASC | DESC]]
  [ROWS | RANGE между ... и ...]
)

    Разбираем каждую часть:

    PARTITION BY — на какие группы разбиваем. Работает как GROUP BY, но без схлопывания. Можно по одной колонке, можно по нескольким: PARTITION BY department, position.

    ORDER BY — порядок строк внутри окна. Нужен для нарастающих итогов, рангов и скользящих средних. Без ORDER BY — окно считается по всей группе сразу.

    ROWS / RANGE — границы окна. Какие строки учитывать: все в группе, только предыдущие 3, только следующие 2 и так далее. По умолчанию — все строки в PARTITION BY.

    Три уровня использования оконных функций¶

    Уровень 1. Сравнить каждую строку со всей группой¶

    AVG(salary) OVER (PARTITION BY department)

    Самый частый случай. PARTITION BY делит таблицу на отсеки (отделы), и для каждой строки считается итог по её отсеку.

    Уровень 2. Сравнить каждую строку со всей таблицей¶

    AVG(salary) OVER ()

    Пустые скобки — окно на всю таблицу. Каждая строка видит общую среднюю.

    Можно комбинировать:

    SELECT
  name,
  department,
  salary,
  AVG(salary) OVER (PARTITION BY department) AS avg_dept,  -- по отделу
  AVG(salary) OVER () AS avg_company,                      -- по компании
  salary - AVG(salary) OVER (PARTITION BY department) AS diff_from_dept
FROM employees;

    Результат — три колонки расчётов рядом с каждой строкой, и ни одна строка не потеряна.

    Уровень 3. Нарастающий итог (про ORDER BY — в следующих уроках)¶

    SUM(revenue) OVER (PARTITION BY year ORDER BY month)

    Здесь ORDER BY month заставляет SUM суммировать не всю группу сразу, а нарастающим итогом: январь → январь+февраль → январь+февраль+март. Эту тему разберём детально в уроке про нарастающие итоги.

    Что важно запомнить про синтаксис¶

    Элемент Обязательный? Что делает
    Функция (SUM, AVG, COUNT...) ✅ Да Что считаем
    Выражение (колонка) ✅ Да По какой колонке считаем
    OVER() ✅ Да Ключевое слово окна
    PARTITION BY ❌ Нет На какие группы делить
    ORDER BY ❌ Нет Порядок строк в окне
    ROWS/RANGE BETWEEN ❌ Нет Какие строки включать

    Если не указать PARTITION BY — окном становится вся таблица. Если не указать ORDER BY — итог считается по всей группе сразу. RANGE BETWEEN по умолчанию — BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING (все строки группы).

    Пример: доля продаж товара в категории¶

    SELECT
  product,
  category,
  revenue,
  revenue * 100.0 / SUM(revenue) OVER (PARTITION BY category) AS share
FROM sales;

    PARTITION BY category — делит продажи на категории. SUM(revenue) OVER (...) — считает общую выручку категории. И мы делим выручку товара на выручку категории — получаем долю.

    Для товара с выручкой 40 000 в категории с общей выручкой 100 000: 40 000 / 100 000 * 100 = 40%.

    Пример: общий итог компании плюс по регионам¶

    SELECT
  region,
  month,
  sales,
  SUM(sales) OVER (PARTITION BY region) AS region_total,
  SUM(sales) OVER () AS company_total,
  ROUND(sales * 100.0 / SUM(sales) OVER (), 1) AS pct_of_company
FROM monthly_sales;

    Здесь сразу три оконных функции в одном запросе: — продажи по региону — продажи по всей компании — доля строки в общих продажах

    Ключевая мысль¶

    Оконная функция = обычный агрегат (SUM, AVG, COUNT) + OVER() + деление на группы через PARTITION BY.

    Главное отличие от GROUP BY: строки не пропадают.

    Мини-задача¶

    У тебя таблица orders:

    order_id | customer_id | amount | date

    Напиши запрос, который для каждого заказа покажет: - сумму заказа - средний чек этого клиента - на сколько заказ отличается от среднего чека клиента (больше или меньше)

    Подсказка 
    SELECT
  order_id,
  customer_id,
  amount,
  AVG(amount) OVER (PARTITION BY customer_id) AS avg_customer,
  amount - AVG(amount) OVER (PARTITION BY customer_id) AS diff
FROM orders;

    Частая ошибка¶

    Путают с GROUP BY.

    GROUP BY Оконная функция
    Строки Схлопываются в одну на группу Остаются как есть
    Результат Меньше строк, чем в исходнике Столько же строк
    Что видишь Только итог по группе Каждую строку + итог рядом
    Когда брать Нужны только итоги по группам Нужны строки + расчёты по группам

    Памятка¶

    • SUM/AVG/COUNT OVER (PARTITION BY колонка) — агрегат по группе для каждой строки
    • OVER () — агрегат по всей таблице
    • PARTITION BY делит таблицу на отсеки, как GROUP BY, но без потери строк
    • Можно использовать несколько оконных функций в одном запросе
    • Синтаксис: <функция>(<колонка>) OVER ([PARTITION BY ...] [ORDER BY ...] [ROWS ...])
    • ORDER BY и ROWS — для нарастающих итогов и скользящих средних (разберём отдельно)
    • Не надо писать подзапросы — одна строка OVER решает
