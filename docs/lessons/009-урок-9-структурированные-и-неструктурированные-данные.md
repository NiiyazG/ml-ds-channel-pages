Типы данных в анализе и ML

Данные бывают разными — и от того, как они организованы, зависит то, какие инструменты и методы мы можем использовать.

───

Структурированные данные

Это данные, которые живут в таблицах — в строках и столбцах. У каждого значения есть чёткое место и тип.

Примеры:

• Таблица клиентов: имя, возраст, email, дата регистрации
• Данные о транзакциях: ID, дата, сумма, товар
• Excel-таблица с продажами

Характеристики:

• Фиксированные поля (столбцы)
• Одинаковая структура для всех записей
• Легко фильтровать, сортировать, агрегировать
• Хранятся в SQL-базах, Excel, CSV

───

Неструктурированные данные

Это данные без заранее определённой формы — их нужно «причёсывать».

Примеры:

• Текст (отзывы, письма, статьи)
• Изображения (фото, скриншоты)
• Аудио (записи разговоров)
• Видео
• JSON-логи

Характеристики:

• Нет фиксированной структуры
• Часто занимают много места
• Требуют специальной обработки

───

Полуструктурированные данные

Золотая середина — данные имеют организацию, но не строгую таблицу.

Примеры: JSON, XML, HTML

───

Что важно понимать

1. Большинство бизнес-задач решаются со структурированными данными
2. Неструктурированные данные — это 80-90% всех данных в мире
3. ML-модели часто требуют превращения неструктурированных данных в структурированные

───

Практический пример

Интернет-магазин:

| Тип данных            | Пример                |
| --------------------- | --------------------- |
| Структурированные     | Таблица заказов       |
| Полуструктурированные | Логи в JSON           |
| Неструктурированные   | Отзывы клиентов, фото |
───

Главное: Уметь превращать сырые данные в таблицу для анализа.
