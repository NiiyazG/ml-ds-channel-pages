# ML. Урок 47 из 101
## Сбор данных

Прежде чем строить модели, графики и выводы — данные надо найти. Это не техническая мелочь, а фундамент всего проекта.

Плохие данные + хорошая модель = плохой результат.

Где брать данные

1. Открытые датасеты: Kaggle, UCI, Data.gov
2. Внутренние БД: ERP, CRM, логи, транзакции
3. API: соцсети, погода, гео, финансы
4. Парсинг: веб-скрапинг (с осторожностью)
5. Ручной сбор: опросы, A/B-тесты, наблюдения

Главные критерии при сборе

Репрезентативность — отражает ли выборка реальную популяцию?
Свежесть — данные за 5 лет могут быть бесполезны
Достоверность — кто и как собирал? Есть ли ошибки?
Полнота — сколько пропусков? Почему?

Ошибка новичка

Взять первый попавшийся датасет и сразу строить модель. Правильно: потрать 80% времени на сбор и подготовку, 20% — на анализ.

Пример из жизни

Хочешь предсказать спрос на товар. Берёшь данные продаж за прошлый год. Если в прошлом году был ковид — данные не репрезентативны. Если нет данных об акциях — модель учится на искажённых цифрах.

Практическое задание

Найди датасет по своей задаче. Выпиши:
— откуда он
— за какой период
— сколько строк и признаков
— какие есть пропуски
— можно ли ему доверять

Типичная ошибка

«Соберу побольше данных — модель сама разберётся». Больше ≠ лучше. Шумные данные учат шуму. Качество важнее количества.

Завтра: очистка данных.
